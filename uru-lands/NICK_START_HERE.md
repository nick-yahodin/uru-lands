# Инструкция для Nick — первый запуск Claude Code

Этот файл читаешь ТЫ, не Claude. Внутри — простые шаги и готовый текст,
который нужно скопировать в Claude Code Desktop.

---

## Шаг 1 — Распакуй ZIP в нужную папку

У тебя в Finder должна быть структура:
```
~/Development/
  └── uru-lands/    ← сюда распакуй содержимое ZIP
      ├── CLAUDE.md
      ├── README.md
      ├── main.py
      ├── ...
```

Важно: если твой основной репозиторий — это `nick-yahodin` monorepo,
то пусть будет:
```
~/Development/nick-yahodin/Development/uru-lands/
```

Проверь один раз: `CLAUDE.md` должен лежать в корне папки uru-lands.
Claude Code читает его автоматически при открытии папки.

## Шаг 2 — Открой папку в Claude Code Desktop

1. Запусти Claude Code Desktop (иконка в Applications)
2. File → Open Folder → выбери папку uru-lands
3. Подожди 5-10 секунд пока Claude Code проиндексирует файлы
4. В нижней части окна появится поле для ввода сообщений

## Шаг 3 — Вставь первый промпт

Скопируй текст между линиями и вставь в поле ввода Claude Code:

---НАЧАЛО ПРОМПТА---

Привет. Я — Nick. Я не разработчик, и операционную работу с кодом,
git, тестами и деплоем делаешь ты. Я даю идеи и направление на
русском, ты отвечаешь на русском, но код, коммиты, комментарии и
документация — на английском.

Это проект uru-lands — агрегатор данных о земельных участках и
аукционах в Уругвае. Прочитай CLAUDE.md в корне — там полный контекст.

Сейчас нужно сделать три вещи:

1. Проверь что структура проекта на месте: запусти ls, убедись что
   есть все модули из CLAUDE.md (config.py, models.py, scraper.py,
   impo_parser.py, impo_scraper.py, enrichment.py, auction_formatter.py,
   telegram_bot.py, duplicate_checker.py, main.py, tests/, .github/).

2. Запусти тесты: python tests/test_impo_parser.py — должно быть
   39/39. Если что-то не работает — почини сам, не спрашивай меня.

3. Настрой git для этой папки:
   - Проверь git status (репозиторий уже существует, я его клонировал)
   - Если нужно, настрой git user.name и user.email
   - Покажи мне текущий remote (git remote -v)
   - Убедись что .env в .gitignore и НЕ попадёт в коммит

После этих трёх шагов — дай краткий отчёт что сработало, а что нет.
Не делай коммит без моего явного разрешения.

---КОНЕЦ ПРОМПТА---

## Шаг 4 — После первого ответа

Когда Claude Code покажет отчёт, ты просто пишешь дальше на русском,
что хочешь. Примеры команд, которые можно давать:

- "Запусти парсер IMPO в dry-run режиме и покажи что он нашёл"
- "Добавь парсер для BuscandoCasa.com"
- "Почему упал тест? Почини"
- "Закоммить всё в git с сообщением 'add IMPO parser'"
- "Задеплой проект на мой сервер" (если настроен SSH)
- "Напиши инструкцию как получить Telegram bot token"
- "Покажи как будет выглядеть пост в Telegram для первого аукциона"

Ты никогда не пишешь команды в терминале сам. Только говоришь Claude
Code, что хочешь — он всё делает.

## Что делать когда застрял

Если Claude Code просит тебя что-то сделать в терминале, а ты не
понимаешь — отвечай: "Сделай это сам вместо меня." В 95% случаев он
может и сделает.

Единственное что Claude Code НЕ может сам:
- Оплатить что-то (нужна твоя карта)
- Авторизоваться в твоих личных аккаунтах (нужны твои пароли)
- Принять юридические решения за тебя

Во всех остальных случаях — он делает сам.

## Что делать если что-то сломалось

Скажи: "Что-то сломалось — разберись и почини." Claude Code прочитает
логи, найдёт проблему, исправит. Не пытайся чинить сам.
